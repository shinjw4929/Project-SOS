## 주석 검토 실행

### 1단계: 변경된 파일 파악

다음 중 하나로 검토 대상 파일을 결정한다:
- `git diff --name-only`로 변경된 파일 목록 확인
- 사용자가 특정 파일을 지정한 경우 해당 파일만 대상

### 2단계: 파일별 주석 검토

각 파일에 대해 다음을 검사한다:

**XML Summary (`/// <summary>`)**:
- 클래스/시스템의 역할 설명이 실제 동작과 일치하는가
- 새로 추가된 기능이 summary에 반영되었는가

**인라인 주석 (`//`)**:
- 주석이 코드 동작을 정확히 설명하는가
- 변경된 로직에 대한 주석이 업데이트되었는가
- 삭제/변경된 동작을 설명하는 주석이 남아있지 않은가

**불필요한 주석**:
- 코드만으로 자명한 동작에 대한 주석이 있는가 (있으면 제거 제안)

### 3단계: 결과 보고

검토 결과를 표로 정리한다:

| 파일 | 상태 | 문제 |
|------|------|------|
| 파일명 | OK / 수정 필요 | 문제 설명 |

### 4단계: 수정

사용자 확인 후 문제가 있는 주석을 수정한다.
